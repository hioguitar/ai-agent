# CLAUDE.md — AI Agent プロジェクト

## 会社概要

- **社名**: （建設会社）
- **売上**: 約10億円
- **従業員**: 10名
- **業種**: 建設業（建設・介護・不動産の複合事業）
- **現状**: 紙・Excelベースで業務を運営中
- **目標**: AIエージェントを駆使して売上100億円を目指す

## このプロジェクトの目的

Google Drive を基盤として、各部署（建設・経理・総務・経営戦略・ITインフラ・介護・不動産）に専門AIエージェントを配置し、それらを統括するスーパーバイザーエージェントと、全体を第三者目線で監視・評価するオブザーバーエージェントを構築する。

## 部署構成

| 部署 | 役割 |
|------|------|
| 建設部門 | 工事管理・施工スケジュール・協力会社管理 |
| 経理部 | 請求・支払・帳票・資金繰り管理 |
| 総務部 | 人事・勤怠・文書管理・庶務 |
| 経営戦略部門 | KPI管理・戦略立案・経営分析 |
| ITインフラ部門 | システム管理・ツール導入・AI運用保守 |
| 介護部門 | 利用者管理・シフト・介護記録 |
| 不動産部門 | 物件管理・契約・入居者対応 |

## エージェント構成

```
[オブザーバーエージェント]  ← 第三者目線で全体を監視・評価
         |
[スーパーバイザーエージェント]  ← 全体統括・タスク振り分け
         |
 ┌───┬───┬───┬───┬───┬───┐
建設  経理  総務  戦略  IT  介護  不動産
```

## 開発方針

- **基盤**: Google Drive（ファイル共有・ドキュメント管理）
- **AIモデル**: Claude API（Sonnet / Opus / Haiku を用途別に使い分け）
- **言語**: Python（Claude SDK）
- **段階的導入**: フェーズ1〜4で段階的に機能拡張
- **コスト意識**: 小規模から始めて効果を確認しながら拡大

## コーディング規約

- コメントは必要最小限（WHYが非自明な場合のみ）
- エラーハンドリングはシステム境界（API・ユーザー入力）のみ
- 各エージェントは独立して動作可能な設計にする
- Google Drive のフォルダ構造をそのままエージェントの管轄範囲とする
